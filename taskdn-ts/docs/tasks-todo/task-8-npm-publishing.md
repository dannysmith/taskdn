# Task 8: npm Publishing

Set up multi-platform builds and npm publishing.

## Multi-Platform Builds

NAPI-RS native bindings need to be built for each target platform:

- `aarch64-apple-darwin` (macOS Apple Silicon)
- `x86_64-apple-darwin` (macOS Intel)
- `x86_64-unknown-linux-gnu` (Linux x64)
- `x86_64-pc-windows-msvc` (Windows x64)

### GitHub Actions Workflow

Create `.github/workflows/ci.yml`:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:

jobs:
  build:
    strategy:
      matrix:
        include:
          - os: macos-latest
            target: aarch64-apple-darwin
          - os: macos-13
            target: x86_64-apple-darwin
          - os: ubuntu-latest
            target: x86_64-unknown-linux-gnu
          - os: windows-latest
            target: x86_64-pc-windows-msvc

    runs-on: ${{ matrix.os }}

    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v2
      - uses: dtolnay/rust-toolchain@stable
        with:
          targets: ${{ matrix.target }}

      - run: bun install
      - run: bun run build -- --target ${{ matrix.target }}

      - uses: actions/upload-artifact@v4
        with:
          name: bindings-${{ matrix.target }}
          path: '*.node'

  test:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v2
      - uses: actions/download-artifact@v4
        with:
          name: bindings-x86_64-unknown-linux-gnu
      - run: bun install
      - run: bun test

  publish:
    if: github.ref == 'refs/heads/main'
    needs: [build, test]
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v2
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          registry-url: 'https://registry.npmjs.org'
      - uses: actions/download-artifact@v4
      - run: bun install
      - run: bun run artifacts
      - run: npm publish --access public
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
```

Note: We use `oven-sh/setup-bun` for bun, but still need `setup-node` for `npm publish`.

## Package Configuration

### package.json updates

```json
{
  "name": "@taskdn/sdk",
  "version": "0.1.0",
  "publishConfig": {
    "access": "public"
  },
  "optionalDependencies": {
    "@taskdn/sdk-darwin-arm64": "0.1.0",
    "@taskdn/sdk-darwin-x64": "0.1.0",
    "@taskdn/sdk-linux-x64-gnu": "0.1.0",
    "@taskdn/sdk-win32-x64-msvc": "0.1.0"
  }
}
```

### Platform packages

NAPI-RS generates platform-specific packages in `npm/` directory. Each needs its own `package.json`.

## Version Management

Options:

1. **Manual** - Update version in package.json, tag, push
2. **Changesets** - Automated versioning and changelogs
3. **Release Please** - Google's automated release tool

For simplicity, start with manual versioning:

```bash
# Bump version (npm version works fine even with bun)
npm version patch  # or minor, major

# Push with tags
git push --follow-tags

# CI publishes automatically on main
```

## Pre-publish Checklist

- [ ] Version bumped appropriately
- [ ] All tests pass
- [ ] README has correct version references
- [ ] `npm pack` produces expected files

## First Publish

```bash
# Login to npm
npm login

# Dry run to verify
npm publish --dry-run

# Actual publish
npm publish --access public
```

## Files to Create

- `.github/workflows/ci.yml` - CI/CD workflow
- `npm/*/package.json` - Platform package configs (generated by napi)

## Notes

- The `@napi-rs/cli` handles most of the platform package complexity
- First publish requires npm account with 2FA
- Consider setting up GitHub Releases alongside npm
- `npm publish` is used even with bun projects (it's the npm registry command)
